# Document

Document the requested repository, area, feature, pattern, or explicit decision from current evidence.

## Intake

- Resolve whether the request is an initial repository pass or a narrow target.
- Confirm the intended scope when the requested subject could map to materially different areas.
- Use current source, tests, configuration, and verified commands as evidence.

Write only within `.ai/docs/**`. Create root documents, child directories, and child indexes only when evidence
justifies them. Do not manufacture historical rationale: create a decision record only from an authoritative record,
explicit user input, or a decision made in an approved workflow.

## Outcome

Produce concise routing and knowledge that reduces future rediscovery. Update the nearest index, identify uncertainty,
and report created or changed paths plus the evidence used. If source or product changes are needed, stop and recommend
a delivery operation rather than making them in this operation.
